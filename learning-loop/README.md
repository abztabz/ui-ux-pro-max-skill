# Learning Loop — Amit Shrestha Website

A self-improving knowledge base for this project. Each work cycle is logged as a
**trace**, distilled into **lesson candidates**, and the proven ones are promoted
into the **brain** (`brain.md`) — which future work reads *before* starting.

Layers: **People** (judgment — what's good, what to promote) · **Agents**
(execution + logging) · **Context** (`brain.md`, the shared memory).

## Files
- `brain.md` — promoted, standing lessons. **Read this before any new work.**
- `traces.md` — append-only log of what happened each cycle. Never edit past entries.
- `lessons.md` — candidate lessons between Distill and Promote, with status.
- `promotions.md` — one line per promotion (what, why, which cycle confirmed it).

## The cycle
Run (read `brain.md`) → Capture (append a trace) → Distill (trace → candidate) →
Promote (human gate + pattern → lesson in `brain.md`) → Verify (still applied? still true?).

## Promotion gates (all six must pass)
1. Pattern, not incident (≥2 occurrences — unless high-severity & obviously correct)
2. Generality 3. Actionability 4. Non-contradiction 5. Cost-of-relearning 6. Ownership
